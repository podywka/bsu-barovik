# Шаг 1: Проектирование справочников

## 1. Справочник городов Беларуси (Cities)

**Описание:** Содержит информацию о городах Беларуси с демографическими и экономическими данными.

**Поля (колонки):**
| Поле | Тип данных | Описание | Обязательное |
|------|------------|----------|---------------|
| id | INTEGER (Primary Key) | Уникальный идентификатор города | Да |
| name | TEXT | Название города | Да |
| region | TEXT | Область (Минская, Гомельская, и т.д.) | Да |
| population | INTEGER | Численность населения (целое число) | Да |
| area | DECIMAL(10,2) | Площадь города в км² (с фиксированной запятой) | Да |
| foundation_date | DATE | Дата основания города | Нет |
| is_industrial_center | BOOLEAN | Является ли промышленным центром | Да |
| description | TEXT (многострочный) | Дополнительная информация о городе | Нет |
| created_at | DATETIME | Дата создания записи | Да |
| updated_at | DATETIME | Дата последнего обновления | Да |

**Типы данных:**
- ✅ Текст: name, region, description
- ✅ Дата: foundation_date, created_at, updated_at
- ✅ Целое число: population
- ✅ Число с фиксированной запятой: area
- ✅ Логический тип: is_industrial_center

## 2. Справочник промышленных предприятий (IndustrialEnterprises)

**Описание:** Содержит информацию о промышленных предприятиях Беларуси с привязкой к городам.

**Поля (колонки):**
| Поле | Тип данных | Описание | Обязательное |
|------|------------|----------|---------------|
| id | INTEGER (Primary Key) | Уникальный идентификатор предприятия | Да |
| name | TEXT | Название предприятия | Да |
| city_id | INTEGER (Foreign Key → Cities.id) | Идентификатор города расположения | Да |
| industry_type | TEXT | Отрасль промышленности | Да |
| employee_count | INTEGER | Количество сотрудников (целое число) | Да |
| annual_revenue | DECIMAL(15,2) | Годовая выручка в BYN (с фиксированной запятой) | Да |
| foundation_year | INTEGER | Год основания предприятия | Да |
| is_state_owned | BOOLEAN | Государственная форма собственности | Да |
| contact_person | TEXT | Контактное лицо | Нет |
| email | TEXT | Электронная почта | Нет |
| phone | TEXT | Телефон | Нет |
| address | TEXT | Адрес предприятия | Да |
| notes | TEXT (многострочный) | Примечания | Нет |
| created_at | DATETIME | Дата создания записи | Да |
| updated_at | DATETIME | Дата последнего обновления | Да |

**Типы данных:**
- ✅ Текст: name, industry_type, contact_person, email, phone, address, notes
- ✅ Целое число: employee_count, foundation_year
- ✅ Число с фиксированной запятой: annual_revenue
- ✅ Логический тип: is_state_owned
- ✅ Дата: created_at, updated_at
- ✅ Внешний ключ: city_id

---

## Зависимость справочников

### Связь между справочниками:
**Города (Cities)** ← **Промышленные предприятия (IndustrialEnterprises)**

- Поле `city_id` в справочнике предприятий является внешним ключом (Foreign Key), ссылающимся на поле `id` в справочнике городов
- Это отношение "один-ко-многим": один город может иметь множество предприятий, но каждое предприятие принадлежит только одному городу

### Бизнес-логика связи:
1. При добавлении нового предприятия обязательно указывается город из справочника городов
2. При удалении города проверяется, есть ли в нем предприятия (запрет или каскадное удаление в зависимости от настроек БД)
3. В интерфейсе при выборе города для предприятия используется выпадающий список, заполненный названиями городов, но хранится `id` города
4. При отображении информации о предприятии вместо `city_id` показывается название города (JOIN запрос)
